# B2B Partnership Recommendation Engine

Matches businesses with strategic partners using LLM-generated reasoning profiles and vector similarity search. Built on local inference via Ollama, with a Go API and Python ML pipeline backed by PostgreSQL + pgvector.

## Prerequisites

- [Ollama](https://ollama.com) running locally with `llama3.1` and `qwen3-embedding` pulled
- PostgreSQL with the `pgvector` extension enabled
- Python 3.10+
- Go 1.21+

## Setup

**1. Database**

```bash
psql -U postgres -f sql/schema.sql
```

**2. Python environment**

```bash
pip install -r requirements.txt
```

**3. Generate synthetic profiles**

```bash
python recommendation/data_gen/generate.py
```

**4. Embed profiles**

```bash
python recommendation/embedding/embed_service.py
```

**5. Run the API**

```bash
go run api/main.go
```

## Architecture

```
Business Profile
      │
      ▼
  llama3.1  ──────────────────────────────────────────────┐
  (reasons about ideal partner characteristics)           │
      │                                                   │
      ▼                                                   ▼
qwen3-embedding                                    Reasoning text
  (1024-dim vector)                                (attached to results)
      │
      ▼
pgvector cosine similarity
  + hard metadata filters
  (location, category, trade_type, roles)
      │
      ▼
  Ranked matches with reasoning
```

The LLM does not embed raw profiles. Each business is first represented as a composite text block (`description + industry + roles + tags + trade_regions`), then embedded. The LLM's role is to reason about partnership fit — the embedding captures that reasoning, not just the business's own attributes.

## Project Structure

```
recommendation/
  data_gen/       generate and validate synthetic business profiles
  embedding/      embed profiles and chunk text for pgvector ingestion
  evaluation/     retrieval quality metrics and test query suite
api/              Go HTTP API — query endpoint, filter handling, result formatting
data/             generated business profiles (JSON)
sql/              pgvector schema and index definitions
```

## Evaluation

```bash
python recommendation/evaluation/eval.py
```

Runs the test query suite in `evaluation/test_queries.json` and reports retrieval metrics against the embedded profile set.

## Notes

- All inference runs locally via Ollama — no external API calls
- Synthetic profiles are generated by `llama3.1` for development since real SME data is not available
- Metadata filters are applied as hard constraints before vector ranking, not as score penalties
